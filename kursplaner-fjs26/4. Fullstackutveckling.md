## KURSENS HUVUDSAKLIGA INNEHÅLL

Denna kurs fördjupar den studerandes förmåga att utveckla kompletta fullstackapplikationer med Node.js i backend och React i frontend där Typescript ska användas end-to-end.

Datalagring behandlas med SQL-databaser och förståelse för ER-diagram, normalisering och relationer mellan tabeller ingår. NoSQL databaser introduceras som ett alternativ. Studenten lär sig att integrera databaskoppling och anrop direkt i kodbasen men därefter använda en ORM (som Drizzle) som abstraktion ovanpå. Containerisering med Docker används som utvecklingsmiljö och för isolering av beroenden mellan kodbas och datalagring.

Studenten lär sig designa och implementera REST-API:er med kontrakter, versionering och felhantering, samt grundläggande backendsäkerhet som autentisering, aktorisering, validering, OWASP-medvetenhet och säker hantering av secrets.

I frontend används React som ramverk. Den studerande lär sig JSX, komponenter, props, hooks och state management — både med Context och externa bibliotek som Zustand, Redux eller motsvarande — samt Reacts renderingsmodell och hur prestandaoptimeringar kan ske. Studenten introduceras också till Reacts ekosystem och till ett fullstackramverk som Next.js.

Studenten ska få inblick i arkitekturprinciperna single responsibility (SRP) och dependency inversion (DIP) i konkret tillämpning av lager­strukturen.

## KURSMÅL

Vid avslutad kurs med godkända resultat, kan den studerande:

**KUNSKAPER:**

1. Redogöra för SQL-databaser och datamodellering med ER-diagram, normalisering
2. Redogöra för NoSQL-database och deras användningsområden i förhållande till SQL
3. Redogöra för REST API och dess användningsområden
4. Redogöra för arkitekturprinciperna single responsibility (SRP) och dependency inversion (DIP)
5. Redogöra för grundläggande backendsäkerhet (autentisering, autorisering, validering, OWASP top 10 och säker hantering av secrets)
6. Redogöra för containerisering med Docker
7. Redogöra för Reacts komponentmodell — JSX, komponenter, props och hooks
8. Redogöra för state management i React — Context och externa bibliotek (t.ex. Zustand eller Redux)
9. Redogöra för Reacts renderingsmodell och vanliga prestandaoptimeringar
10. Redogöra för Reacts ekosystem — routing (t.ex. React Router), fullstackramverk (t.ex. Next.js), styling-strategier och vanliga komponentbibliotek

**FÄRDIGHETER:**

11. Implementera backend-tjänster med Node.js och Typescript enligt routes → controllers → services-arkitektur
12. Modellera och implementera SQL-databaser (PostgreSQL eller motsvarande) — med både raw SQL skrivet i kod och med en ORM (t.ex. Drizzle)
13. Implementera NoSQL-datalagring (MongoDB eller motsvarande) där det är lämpligt utifrån krav
14. Designa och implementera REST-API:er med autentisering, autorisering, validering och felhantering
15. Använda Docker för utvecklingsmiljö och containerisering av backend-tjänster och deras beroenden
16. Utveckla en frontend-applikation i React — komponenter, hooks, state management (Context och externa bibliotek), routing och prestandamedveten utveckling

**KOMPETENSER:**

17. Självständigt driva utveckling av en backendtjänst — från datamodellering och API-design till levererad och containerisierad applikation
18. Självständigt driva utveckling av en React-frontend — från komponentdesign och state management till prestandaoptimerad och levererad applikation

Betyg sätts i form av icke godkänt (IG), godkänt (G) eller väl godkänt (VG)

**Icke godkänt (IG):** Samtliga kursmål är inte uppfyllda

**Godkänt (G):** Samtliga kursmål är uppfyllda

**Väl godkänt (VG):** Den studerande har genomfört kursen och uppnått godkänt i alla kursmål. Dessutom kan den studerande med stor säkerhet och skicklighet:

- Självständigt driva utveckling av en backend-tjänst — från datamodellering och API-design till levererad och containerisierad applikation (kursmål 17)
- Självständigt driva utveckling av en React-frontend — från komponentdesign och state management till prestandaoptimerad och levererad applikation (kursmål 18)
